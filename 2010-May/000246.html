Hi Bill,
I had a look at the diffs between the gcm branch and trunk and I have a 
few comments/suggestions:
1. you don't need to define special double precision variables for the 
hot file, i.e. all the [*_hot] variables in glide_vars.def, etc. For a 
hot file you just set the default precision to double, e.g.

[CF output]
name: hot.nc
frequency: 1000
variables: hot
xtype: double

2. there are still quite a few blocks of code marked
    !lipscomb
    I think you should remove those markers, svn can handle that for us
    just use svn blame/praise

3. the code is also still instrumented with debugging messages, if they 
are generally useful, rather than removing them, I'd conditionally 
compile them with
#ifdef DEBUG_WHATEVER

#endif blocks


I haven't really looked at the modifications of libglint so I won't 
comment on these.
