Hi all,

This may be of interest to those of you working on merging new code  
into the glimmer-cism2 trunk.

As you may have noticed, I just checked in a bunch of CCSM-related  
changes to glimmer-cism-gcm (a branch on the glimmer-cism2 trunk).   
More changes will follow, but this is the main batch.  In a few weeks  
the branch will be tagged as Glimmer-CISM 1.6, frozen (except for bug  
fixes), and released as part of the Community Earth System Model.

At any point, I can check these changes into the glimmer-cism2 trunk.   
I'd like to do this before those of you working on glimmer-cism-lanl  
start merging code into glimmer-cism2.  Please let me know if and when  
you're ready, and I'll commit my changes to the trunk.  Most of the  
additional changes I make will be in Glint and will be independent of  
the higher-order code.

Jesse, Steve et al.:  If you have glimmer-cism-lanl code ready for the  
CESM release, please start checking it into the glimmer-cism-gcm  
branch and testing it.  This could include some of the test suites we  
talked about (but not the HO solvers, of course).

Ian and Rupert: You might want to look at the changes I've made in  
Glint to accommodate CCSM coupling fields.  The interfaces are a bit  
messy in that I've added optional CCSM-related arguments to various  
glint routines instead of writing new CCSM-specific routines.  (The  
only totally new subroutines are in glint_global_interp.F90.)  This  
approach seemed better than writing new routines and having a lot of  
duplicate code.  However, the code could be much cleaner.  Please let  
me know if you have ideas on how to improve things.

Any comments and suggestions are welcome.

Regards,

Bill
