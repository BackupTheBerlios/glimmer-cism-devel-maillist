> Hi all,

> 
> At the start of most .F90 files there is something like ...
> 
> #include "glide_nan.inc"
> #include "glide_mask.inc"
> 
> ... for example, at the head of glide_velo_higher.F90.
> 
> I recently made some seemingly trivial changes to that file so that some 
> options could be specified at run time rather than being hardwired into 
> the code. When I did this, I got a build error that I have not seen before:
> 
> *** No rule to make target `config.inc', needed by 
> `glide_velo_higher.o'.  Stop.
> 
> I fixed this by adding:
> 
> #include "config.inc"
> 
> to the head of glide_velo_higher.F90. This also required that I comment

>

> this and other parts of the code before but have never seen this error 
> come up.

Hi Steve,
how did you run configure? the macro HAVE_CONFIG should be defined after
you have run configure.  When you run make what options are passed to
the compiler? You should see something like -DHAVE_CONFIG.

Is there a bug in glide_nan.inc or glide_mask.inc? These should also
come after including config.inc.
