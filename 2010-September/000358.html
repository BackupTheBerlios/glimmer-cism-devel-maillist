> default we are essentially presenting a model that is 20 years (well 15) 
> old, which may not be good PR.  i am not certain that we can tackle this 
> separately from the dycore issue in that whatever solution we choose

yes, I am sure the HO code will become the default. at the moment it is 
more a pragmatic question of where to put it before the refactoring of 
the code, given that we plan to rearrange most things in the not too 
distant future. my understanding of the HO code as it is at the moment, 
is that it requires the existing SIA code to be compiled as well. If 
that was the case then the HO code would be an add-on.

I would expect the code to move somewhere else once we are tackling the 
refactoring and it would become a real alternative to the SIA code.
